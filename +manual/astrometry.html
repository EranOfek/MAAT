The astrometry function

in the astronomy & astrophysics toolbox for MATLAB

Description:

The astrometry function is a method in the SIM class that look for the astrometric solution of astronomical images. The attempt to find matches between stellar patterns in the image and in a reference catalog, and use it to fit an astrometric solution to the image. The code put emphesize on robustness and to find solutions which are good over the entire field of view. The code is described in Ofek et al. (2018).

This file is accessible through the manual package (i.e. manual.mextractor).

Examples

First, read the FITS images into a SIM object:

S = FITS.read2sim('PTF_20140311*.fits');

S = S(1);

The astrometry command can run on multiple images, but here for simplicity we use a single image.

You can start by extracting sources in the image using (e.g., mextractor or ImUtil.Im.sextractor). Note that this step is optional and if the catalog is not provided in the SIM object (S), then astrometry will execute mextractor (or sextractor; see input parameters):

Next you can execute the astrometry

[R, S] = astrometry(S,'Flip',[1 -1;1 1; -1 -1]);

Warning: Units are not provided in catalog - assume units are correct

The first input argument is alawys a SIM object (astrometry is a SIM method). The SIM object is followed by pairs of keyword, value. In this case the 'Flip' keyword indicate which coordinates flip should be attempted. To see a full list of keywords and their default value type

help astrometry

The second output argument (S) is the input SIM object with the header updated with the new WCS keywords containing the astrometric solution. To see this you can display the image in ds9:

ds9(S)

semilogy(R.RefMag(R.FlagG),R.Resid(R.FlagG).*3600,'.')

Note the 'FlagG' field contains a vector of logical indicating the stars used in the solution (after sigma clipping). By default all the stars vectors (e.g., 'Resid', 'RefMag', 'RefColor', ...) contains all the stars and not only the stars used in the solution.
